# REST API Translation Management - Summary

## Что создано

Полноценный навык для автоматического управления переводами REST API ключей в MikoPBX.

```
    ├── sync_translations.py        # Скрипт 3: Синхронизация
    ├── manage_translations.sh      # Интерактивная оболочка
    └── extracted_keys.json         # Результат (генерируется)
```

## Возможности

### 1. Извлечение ключей (extract_keys.py)
- Сканирует все PHP файлы в `/src/PBXCoreREST`
- Находит все использования rest_* ключей
- Определяет контекст (ApiOperation, ApiResponse, etc.)
- Сохраняет результат в JSON

**Статистика:**
- Файлов просканировано: 479
- Ключей найдено: 2876
- Уникальных ключей: 1589

### 2. Валидация (validate_translations.py)
- Сравнивает ключи из кода с ключами в RestApi.php
- Находит отсутствующие ключи (в коде, но не в файле)
- Находит неиспользуемые ключи (в файле, но не в коде)
- Группирует ключи по префиксам для лучшей читаемости

**Текущее состояние:**
- ✅ Валидные ключи: 1370/1589 (86%)
- ❌ Отсутствующие: 219
- ⚠️ Неиспользуемые: 358

### 3. Синхронизация (sync_translations.py)
- Добавляет отсутствующие ключи в RestApi.php
- Удаляет неиспользуемые ключи
- Создает резервную копию перед изменениями
- Валидирует PHP синтаксис после изменений
- Режим dry-run для предварительного просмотра

**Безопасность:**
- Автоматический backup
- Подтверждение перед удалением
- Валидация синтаксиса
- Откат при ошибках

### 4. Интерактивная оболочка (manage_translations.sh)
- Меню с 7 опциями
- Цветной вывод
- Поддержка command-line и интерактивного режимов
- Автоматический workflow (extract → validate → sync)

## Типы обрабатываемых ключей

1. **API Descriptions** - `rest_{Resource}_ApiDescription`
2. **Operation Summaries** - `rest_{abbr}_{Operation}`
3. **Operation Descriptions** - `rest_{abbr}_{Operation}Desc`
4. **HTTP Responses** - `rest_response_{code}_{type}`
5. **Security Schemes** - `rest_security_{type}`
6. **Parameters** - `rest_param_{name}`
7. **Schemas** - `rest_schema_{name}`
8. **Tags** - `rest_tag_{name}`

## Примеры использования

### С Claude Code
```
Пользователь: "Проверь переводы REST API"
Claude: Использует навык для extraction → validation → показывает результаты
```

### Из командной строки
```bash
# Интерактивный режим
./manage_translations.sh

# Command-line режим
./manage_translations.sh validate
./manage_translations.sh sync --add-missing
./manage_translations.sh all
```

### Из Python
```bash
python3 extract_keys.py
python3 validate_translations.py
python3 sync_translations.py --add-missing --dry-run
```

## Workflow интеграция

### При создании нового endpoint
1. Написать controller с REST API attributes
2. Запустить навык через Claude: "Check REST API translations"
3. Добавить отсутствующие ключи: "Add missing translation keys"
4. Перевести placeholder текст
5. Commit изменений

### Периодическое обслуживание
1. Раз в месяц: валидация переводов
2. Проверка неиспользуемых ключей
3. Удаление устаревших (с осторожностью)

### Перед релизом
1. Полная синхронизация
2. Проверка placeholder текста
3. Валидация результата
